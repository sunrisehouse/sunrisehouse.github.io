자료구조

  1. Array and Linked List
  • 공통점: 선형 데이터 저장
  • 차이점: 구현, 특정인덱스조회-삽입-삭제 시간복잡도
  • 대답어레이와 링크드 리스트는 둘 다 선형적인 데이터를 저장하는 자료 구조입니다. 하지만 그 선형적인 자료 구조를 구현 하는데 차이가 있습니다. 어레이는 그 크기만큼 메모리에 연속적으로 할당합니다. 하지만 링크드 리스트는 데이터가 추가 될 때마다 새 메모리를 할당해서 기존 리스트에 연결을 추가합니다. 어레이는 각 데이터들이 연속적인 메모리에 올라가기 때문에 연속적인 주소값을 갖고있습니다. 그래서 특정 인덱스의 접근하려고 할 때 그 주소를 바로 구해서 접근할 수 있습니다. 하지만 링크드 리스트는 연결된 데이터들을 이동하면서 탐색해야 합니다. 특정 인덱스에 삽입 삭제 해야할 경우 어레이는 그 뒤 인덱스의 데이터들을 뒤로 밀어야하기 때문에 O(n) 의 시간이 걸립니다. 하지만 링크드 리스트는 해당 인덱스 앞의 연결을 끊고 새 노드를 연결하기만 하면 돼서 O(1) 의 시간이 걸립니다.
  1. Stack and Queue
  • 공통점: Array 와 List 를 이용해서 선형 적인 데이터를 저장하지만 그 데이터의 삽입 삭제에 규칙이 있는 자료 구조
  • 차이점: LIFO-FIFO, 조회-삽입-삭제 시간 복잡도
  • 대답
  1. Graph
  • 정의: 노드들 간의 관계에 관한 데이터를 저장하는 구조
  • 구현 (인접 행렬, 인접 리스트)
  • 장단점: 인접 행렬 (메모리 비효율, 노드 사이 연결성, 간선 많은 그래프)
  • 장단점: 인접 리스트 (메모리 효율, 노드 사이에 연결되어있는 모든 노드들, 간선 적은 그래프)
  1. Tree
  • 정의: 위계 관계를 저장하기 위한 데이터 구조
  • 구현: 인접 행렬, 인접 리스트, 노드 연결해서
  • 종류
  1. heap
  • 정의: 최댓값, 최솟값 빠르게 찾기 위한 트리
  • 형태: 부모노드 key 값이 자식 노드 key 값 보다 크다.
  • 구현: Array 이용 - index 0 비우고 왼쪽 자식 = 자기 * 2 오른쪽 자식 = 자기 * 2 + 1
  • 시간 복잡도
  • 조회: 트리 높이 = log(n)
  • 추가: 트리 높이 = log(n) = leaf 마지막에 추가하고 부모랑 계속 비교
  • 삭제: 트리 높이 = log(n) = root 제거, 마지막 leaf 를 루트로 큰 자식이랑 비교
  1. Binary Search Tree
  • 정의: 이분 탐색의 원리를 이용해 탐색 빠르게 할 수 있는 Tree 구조
  • 형태: 왼쪽 자식 < 부모 < 오른쪽 자식
  • 조회: O(nlogn) or O(n^2. (=> balance 가 깨진 경우)
  • 삽입: 부모에서 부터 계속 key 값 비교하면서 leaf 까지 내려온다.
  • 삭제: 삭제할 노드 찾고 자식 있는지 없는지 나눠서 진행
  • 중복 X -> 효율 안좋아
  1. AVL Tree
  • Balance Factor 를 삽입 삭제시에 검사해서 좌우 서브 트리의 높이를 맞춘다.
  • Balance Factor 만 허용 +1 , 0, -1
  1. Red Black Tree
  • 정의: 노드에 Red, Black 색을 할당해서 특정 조건 맞추도록 해서 Balance 맞추는 트리
  • Root 는 Black / leaf node 에 연결 안된 node 들 null node 들 black / 새로 추가되는 노드 red / red 자식은 무조건 black
  1. B Tree
  • 정의: 한 노드에 여러 키(M 개)와 여러 자식, balance 맞춤
  • 조회 삽입 삭제 시간복잡도: O(logn)
  • 삽입 삭제: 노드가 최소 키 수(M/2 - 1.와 최대 키 수(M -1. 유지 하도록 보장 작거나 커지면 노드 분리 or 부모랑 합쳐짐
  1. B + Tree
  • 데이터 위치: B Tree 는 모든 노드들에 B+Tree 는 Leaf node 들에만
  • 모든 데이터 탐색: 선형적으로 가능
  • 범위 탐색 유리
  1. Hash
  • 정의: 해쉬 함수를 이용해 데이터 조회, 삽입, 삭제를 O(1. 만에 할 수 있는 자료구조
  • 문제: 충돌 - 해시 테이블 한정 but input 무한정
  • 해결: chaining : 충돌 나면 링크드 리스트로
  • 같은 곳에 계속 충돌 나면 성능 저하 선형적으로
  • 해결: open addressing: 충돌나면 다음 주소로 (바로 다음 or 제곱 다음 or 해시 한번 더)
  • 추가 메모리 필요X
